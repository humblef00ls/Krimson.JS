Guide to Documentation

Every compotent / feature will have its dedicated page with all relevant code markup and class names written in blocks like this All variations of the same feature / component will be mentioned on the same page. However, since my focus is easy and full customisability, I encourage to make some of your own classes for your primary styles.


However, due to a lot of the JQuery generated elements and their very specific configurations, there are CSS variables for quick and easy configuration throughout. They too are in code blocks and all have the same prefix --like-this All varaible names generally explain what they control, and all of them can be found in the :root element of the CSS file. Since I am nice, to change a variable you could just use this funciton setVaraible(varName, newValue) I will mention them next to a component if they are relevant but check :root for a full list. CSS variables are amazing because you can override them differently in different scopes, making customisability much easier in terms of configuring not only basic CSS values but also JQuery added components in SANGUINAR.


The use of square brackets is to signify use of your own value of some kind, like while assigning a unique ID to tabs, like this : [yourIDHere]

If you ever see the icon next to a code block, it means that SANGUINAR generates more dom elements with the script.js file before rendering so in case you're trying to heavily modify the structure or class keep that in mind.

TABS

General Guide

Main tab bar is a grid which dynamically sizes the tabs to fit, which have a min-width defined by --tabs-min-width : 100px and all tabs are flex displays. Minimum of one tab is required.

The slider is added to the first tab and dynamically resizes and moves with width of the first tab. The active tab has the class .tabs-active and slider has .select-rail If you however just want to change the color use --tabs-highlight : 255, 109, 121 Note that this color variable can only be written as 3 numbers to be a part of an RGB value. Another general configuration is the visibility of the colored slider with the use of the variable --tabs-rail-display : block

The tabs and their window are linked by adding a postfix to the ID of the tabs menu bar and making that the ID of tabs window like this : [yourIDHere]-window

The tabs correspond to the windows by their index, i.e the 3rd tab will display the 3rd div in the corresponding window.

You will see below that tabs can be made disabled. They still require some kind of div with the .tabs-view class and will automatically hide their content from the webpage before rendering and all that is done by simply adding a class to the tab. However note that in case you want to enable a disabled tab, you would also have to reload the page as the full script only runs on $(document).ready()

PARALLAX

General Guide

While there are CSS ways to do parallax, I found them not practical, so these are all JQuery based. They hide overflow and in the back scale up the element to be parallaxed. To set up, you firstly need to add class .parallax-listner to the body or the local element inside which you want to detect scroll. This basically adds a JQuery scroll event to that element.

Inside the element, set up the elements to be parallaxed as such : 

 
    

        [yourBackground]
    

    

        [yourContent]
    




--parallax-speed : 7;
--parallax-scale : 1.5;
--parallax-height : 90vh;

Fire in a bottle
Inside .parallax-background and .parallax-base you really can put anything you want. In the demo here I have just a custom style class for the base/foreground and an img element in the background. As you can see, the speed and the scale of the background are easy to change with variables. The height for the element has to be specified with a value and can either be changed by the variable or just override the .parallax class. Note that while --parallax-height works with local scopes, --parallax-speed and --parallax-scale are global only.
Lightning in a bottle
The elements inside .parallax-background get their position offset by the equation ( elpos - center ) / $parallaxspeed where elpos is the element's position in the window, and center is just half of the window's height. $parallaxspeed is whatever value defined in --parallax-speed in the :root of the main style class.
Star in a bottle
