Guide to Documentation

Every compotent / feature will have its dedicated page with all relevant code markup and class names written in blocks like this All variations of the same feature / component will be mentioned on the same page. However, since my focus is easy and full customisability, I encourage to make some of your own classes for your primary styles.

However, due to a lot of the JQuery elements and their very specific configurations, add CSS vasiables throughout. They too are in code blocks and all have the same prefix --like-this All varaible names generally explain what they control, and all of them can be found in the body class of the CSS file. Since I am nice, to change a variable you could just use this funciton setVaraible(varName, newValue)
